================================================================================
INDEX SUMMARY
================================================================================

High-Value Indexes (Create First):
- fsms.search_vector (GIN) - Full-text search
- fsms.definition (GIN) - JSONB queries
- fsms(visibility, is_optimized, created_at) - Browse queries
- algorithm_results(original_fsm_id, algorithm, improvement_percentage)

Medium-Value Indexes:
- All foreign keys (automatic with references)
- fsms.tags (GIN) - Tag filtering
- fsms.created_by - User FSMs
- comments.fsm_id - Comment fetching
- votes.fsm_id - Rating aggregation

Partial Indexes (Condition-Based):
- fsms WHERE visibility = 'public' AND is_optimized = TRUE
- algorithm_results WHERE success = TRUE
- states WHERE is_dummy = FALSE


================================================================================
SECURITY CONSIDERATIONS
================================================================================

Row-Level Security (RLS) Policies:
- fsms: Users see own FSMs + public FSMs + shared FSMs
- comments: Anyone can read public, only authors can edit/delete
- votes: Users can only vote once per FSM per vote_type
- shares: Only share creator can modify

Field-Level Security:
- users.password_hash: Never expose in queries
- users.email: Only visible to self and admins
- shares.password_hash: Never expose
- users.api_key: Only visible to owner

Rate Limiting (Application Layer):
- API: 100 requests/hour per user (configurable)
- FSM creation: 10/hour per user
- Comments: 30/hour per user
